Setup Proyek & Inisialisasi Framework

Mempersiapkan lingkungan kerja, inisialisasi proyek Next.js, instalasi library, dan konfigurasi sistem warna Ayumi Beauty House. Dimulai dari menulis prompt pertama ke Antigravity AI.

🤖

1.1 — Menulis Prompt Pembuka ke Antigravity AI

Mendeskripsikan kebutuhan proyek secara lengkap kepada AI agar dapat membantu pembuatan struktur awal
▾
💡 Tips Menulis Prompt yang Efektif Jelaskan proyek dengan detail: nama usaha, jenis layanan, jumlah pengguna, fitur utama yang dibutuhkan, dan teknologi yang ingin digunakan. Semakin spesifik prompt Anda, semakin tepat saran yang diberikan Antigravity.

Saya ingin membuat aplikasi SaaS manajemen klinik kecantikan bernama "Ayumi Beauty House". Aplikasi ini digunakan oleh tim staf internal (bukan pelanggan umum). Ada 3 tipe pengguna:

  • Owner — pemilik, bisa lihat semua data semua cabang
  • Admin (per cabang) — kelola operasional harian di satu cabang
  • Terapis — input catatan rekam medis pasien setelah tindakan

Fitur utama yang dibutuhkan:

  • Booking antrean pasien dan status kedatangan
  • Rekam medis SOAP (Subjective, Objective, Assessment, Plan)
  • POS Kasir (checkout, diskon, stok produk)
  • Paket kupon sesi multi-visit
  • Notifikasi realtime antar-staf
  • Laporan keuangan dan performa terapis

Tolong buat proyek Next.js App Router dengan Supabase sebagai database dan Tailwind CSS untuk styling. Mulai dari inisialisasi proyek dan struktur folder!

📋 Yang Akan Dilakukan Antigravity Antigravity akan membaca context ini lalu membuatkan struktur folder Next.js, menginstal dependencies yang diperlukan, dan menyarankan arsitektur database awal yang sesuai dengan kebutuhan Anda.

2.2 — Mengambil API Keys Supabase ke .env.local

Menghubungkan Supabase ke project Next.js melalui environment variables
▾
  1. Di Supabase Dashboard → Buka menu Settings (ikon gear di sidebar kiri bawah)
  2. Pilih sub-menu "API"
  3. Salin nilai dari "Project URL" dan "anon public" key
  4. Untuk service_role key (admin): scroll ke bawah ke bagian "Service Role Key" (gunakan hanya di server-side/backend)
Buat file .env.local di root project Anda
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxx.supabase.co NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9... SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
⚠️ Keamanan Penting File .env.local TIDAK boleh di-commit ke GitHub. Pastikan .env.local sudah terdaftar di file .gitignore Anda. NEXT_PUBLIC_ prefix hanya untuk variabel yang aman dibaca browser. SERVICE_ROLE_KEY hanya digunakan di server-side dan scripts!

2.3 — Mendesain Skema Tabel Database (SQL Editor)

Meminta Antigravity membuatkan SQL untuk semua tabel, lalu menjalankannya di Supabase SQL Editor
▾

Tolong buatkan skrip SQL untuk membuat semua tabel database yang dibutuhkan oleh Ayumi Beauty House di Supabase PostgreSQL. Tabel yang diperlukan:

  • branches — data cabang fisik klinik
  • users — staf klinik dengan kolom role (owner/admin/therapist)
  • patients — data pasien beserta status CRM
  • appointments — booking jadwal & status antrean
  • treatment_records — rekam medis SOAP
  • treatment_record_items — daftar tindakan per sesi
  • treatments & treatment_categories — master layanan
  • products & product_stock — produk & inventaris per cabang
  • transactions & transaction_items — catatan kasir
  • coupon_packages, patient_coupons, dll — sistem kupon
  • notifications — sistem pesan realtime antar-staf

Gunakan UUID sebagai primary key, dan buat relasi foreign key yang sesuai antar tabel.

  1. Terima skrip SQL dari Antigravity
  2. Buka Supabase Dashboard → pilih menu "SQL Editor" di sidebar
  3. Klik tombol "+ New query"
  4. Paste skrip SQL dari Antigravity ke editor
  5. Tekan tombol "Run" (atau Ctrl+Enter)
  6. Verifikasi tabel berhasil dibuat di menu Table Editor
🔒

2.4 — Mengaktifkan Row Level Security (RLS) & Membuat Policies

Meminta Antigravity buatkan skrip migrasi RLS lalu jalankan di Supabase
▾

Tolong buatkan skrip SQL migrasi untuk mengaktifkan Row Level Security (RLS) di semua tabel utama. Aturan akses yang diinginkan:

  • Owner: Full access (SELECT, INSERT, UPDATE, DELETE) di semua tabel, semua cabang
  • Admin: SELECT, INSERT, UPDATE di tabel operasional, tetapi hanya untuk data dari branch_id cabang mereka sendiri. Tidak bisa hapus data
  • Therapist: Hanya SELECT di master data (treatments, categories). INSERT & SELECT di rekam medis miliknya sendiri. TIDAK bisa melihat harga atau data transaksi keuangan
📍 Di mana menjalankan skrip migrasi? File migration_all_rls.sql yang dibuat Antigravity dijalankan di Supabase Dashboard → SQL Editor. Anda juga bisa simpan skrip tersebut di folder scratch/ di dalam proyek sebagai dokumentasi permanen.
Contoh Policy RLS yang Dibuat Antigravity
-- Aktifkan RLS pada tabel utama ALTER TABLE public.treatment_records ENABLE ROW LEVEL SECURITY; ALTER TABLE public.appointments ENABLE ROW LEVEL SECURITY; -- Policy: Therapist hanya bisa SELECT rekam medis miliknya CREATE POLICY "Therapist can view own records" ON public.treatment_records FOR SELECT TO authenticated USING ( performed_by = auth.uid() OR EXISTS ( SELECT 1 FROM public.users WHERE users.id = auth.uid() AND users.role IN ('owner', 'admin') ) );
👤

2.5 — Membuat User Pertama (Owner) di Supabase Auth

Mendaftarkan akun owner pertama langsung dari Supabase Dashboard
▾
  1. Buka Supabase Dashboard → pilih menu "Authentication"
  2. Klik tab "Users" → Klik tombol "Add User"
  3. Masukkan email dan password untuk akun Owner pertama → Klik "Create User"

5.1 — Membangun Halaman POS Kasir Utama

Checkout terintegrasi dengan data SOAP terapis, diskon, dan stok otomatis
▾

Tolong buat halaman POS di app/kasir/page.js dengan alur kerja:

Supabase Realtime · Toast · WebSocket

Notifikasi Realtime Antar-Staf

Mengintegrasikan Supabase Realtime channel agar staf (Admin dan Owner) mendapatkan notifikasi pop-up instan di browser ketika ada aktivitas penting — tanpa perlu reload halaman.

🔔

7.1 — Mengaktifkan Supabase Realtime & Membuat Listener

Aktifkan realtime di Supabase lalu buat listener di GlobalHeader
▾
  1. Buka Supabase Dashboard → Database → Replication
  2. Di bagian "Source", pastikan publikasi supabase_realtime aktif
  3. Tambahkan tabel notifications dan appointments ke publikasi realtime melalui SQL Editor:
SQL — Aktifkan Realtime untuk tabel notifications
ALTER PUBLICATION supabase_realtime ADD TABLE public.notifications;

Tolong tambahkan sistem notifikasi realtime ke components/GlobalHeader.js. Gunakan Supabase Realtime untuk mendengarkan INSERT baru di tabel notifications yang memiliki recipient_id sama dengan user yang sedang login.

Ketika ada notifikasi baru, tampilkan toast pop-up (menggunakan react-hot-toast) dengan:

  • Ikon dan warna yang berbeda sesuai tipe notifikasi
  • Tipe patient_arrived: warna hijau, bunyi "Pasien [nama] telah tiba"
  • Tipe therapist_ready: warna biru, bunyi "Terapis siap di ruangan [X]"
  • Notifikasi hanya untuk Admin Cabang (di cabangnya sendiri) dan Owner (semua)
  • Terapis tidak menerima notifikasi jenis ini

Deploy ke Vercel (Production)

Menghubungkan repositori GitHub ke Vercel untuk deployment otomatis, mengatur environment variables production, dan memastikan aplikasi berjalan dengan benar di domain publik.

🚀

8.1 — Import Proyek GitHub ke Vercel

Menghubungkan repositori GitHub ke platform deployment Vercel
▾
  1. Buka https://vercel.com → Login menggunakan akun GitHub Anda
  2. Di dashboard Vercel, klik tombol "Add New..." → pilih "Project"
  3. Vercel akan menampilkan daftar repositori GitHub Anda → cari dan pilih ayumi-beauty-house
  4. Klik tombol "Import"
  5. Vercel secara otomatis mendeteksi framework Next.js → biarkan pengaturan default
  6. JANGAN klik "Deploy" dulu! Anda harus mengatur Environment Variables terlebih dahulu (langkah 8.2)
⚠️ Penting Sebelum Deploy Vercel tidak membaca file .env.local dari GitHub (karena sudah masuk .gitignore). Anda HARUS menginput ulang semua environment variables secara manual di dashboard Vercel sebelum melakukan deploy pertama.
⚙️

8.2 — Mengatur Environment Variables di Vercel

Input ulang semua variabel lingkungan Supabase ke dashboard Vercel
▾
  1. Di halaman import project Vercel, scroll ke bagian "Environment Variables"
  2. Klik "Add" untuk setiap variabel berikut:
⚙️ Environment Variables yang Harus Diinput di Vercel
NEXT_PUBLIC_SUPABASE_URL https://xxxxxxxxxx.supabase.co All Environments
NEXT_PUBLIC_SUPABASE_ANON_KEY eyJhbGci... (key dari Supabase API settings) All Environments
SUPABASE_SERVICE_ROLE_KEY eyJhbGci... (key rahasia, hanya server-side) Production Only
🚫 Keamanan SUPABASE_SERVICE_ROLE_KEY Set scope SUPABASE_SERVICE_ROLE_KEY ke Production Only saja di Vercel. Jangan expose ke Preview atau Development environment. Key ini memberikan akses penuh melewati semua RLS policy!
🌐

8.3 — Deploy & Verifikasi Aplikasi Production

Klik Deploy dan pastikan aplikasi berjalan normal di URL Vercel

  3. Jika build berhasil, Vercel akan memberikan URL production seperti: ayumi-beauty-house.vercel.app
  4. Buka URL tersebut di browser → pastikan halaman login muncul dengan benar
  5. Coba login dengan akun Owner yang sudah dibuat di Supabase Auth

8.4 — Mengatur Custom Domain (Opsional)

Mengganti URL vercel.app dengan domain bisnis sendiri
▾
  1. Di Vercel Dashboard → buka project → pilih tab "Settings"
  2. Klik menu "Domains"
  3. Masukkan domain yang Anda miliki, contoh: app.ayumibeautyhouse.com
  4. Ikuti instruksi Vercel untuk mengatur DNS records di registrar domain Anda (Niagahoster, Namecheap, GoDaddy, dll.)
  5. Tambahkan record CNAME yang mengarah ke cname.vercel-dns.com
  6. Tunggu propagasi DNS (biasanya 5 menit - 24 jam)
  7. Vercel akan otomatis menerbitkan SSL certificate gratis via Let's Encrypt untuk domain Anda
📧

8.5 — Konfigurasi Supabase Auth Redirect URL untuk Production

Update allowed URLs di Supabase agar auth bekerja di domain production
▾
⚠️ Langkah Ini Sering Terlewat! Jika tidak dilakukan, Supabase Authentication akan menolak redirect dari domain production Anda dan login akan gagal.
  1. Buka Supabase Dashboard → pilih menu Authentication
  2. Klik tab "URL Configuration"
  3. Di field "Site URL", ubah dari localhost ke URL production Anda: https://ayumi-beauty-house.vercel.app
  4. Di bagian "Redirect URLs", tambahkan semua URL yang diizinkan:
    https://ayumi-beauty-house.vercel.app/**
    http://localhost:3000/** (untuk development)
